Bulk recruitment

V4


Aim

V4 looked at how a campaign might use a pre-sift (otherwise known as an 'initial sift') and how we could capture it's requirements in the 'create a vacancy' journey. Our aim was to:

  • give an option to allow a pre-sift if necessary
  • capture what and how the pre-sift should be
  • capture what should happen to successful pre-sifted applications
  • show the pre-sift info replicated in the actual job ad
  • show the new pre-sift tasks created when managing the vacancy

What we did

We added a pre-sift option to the existing 'create a vacancy' journey, accessed by clicking on 'Select assessment methods for sift'. We also added an updated task list where new pre-sift tasks could be seen (although they didn't function and weren't tested). Lastly, we added a relevant version of the job ad so the user could see a pre-sift message within the application process section.

V3


Aim

V3 stepped away from the 'create a vacancy' process and moved onto setting up multiple sift panels during bulk campaigns. Our aim was to:

  • demo a vacancy management dashboard, where a user could see vacancies, manage vacancy users, read guidance etc
  • allow a user to get messages / warnings related to bulk campaign activity
  • allow a user to set up both a single and multiple sift panels, including allocating roles to members of the panel

What we did

We created a new prototype for a vacancy management dashboard, which would lead the user onto an updated task list where new sift panels could be added.

V2


Aim

V2 was an iteration of the initial V1 prototype. Based on adding multi-grade / contract functionality and some improvements based on user feedback from V1, our aim was to:

  • allow a user to build a single ad with multiple job types e.g. different grades or contract types
  • increase contact options by offering a 'team mailbox' option
  • amending content where issues had been raised during V1 testing

What we did

We made the above changes to create a V2 version of 'create a vacancy'.
